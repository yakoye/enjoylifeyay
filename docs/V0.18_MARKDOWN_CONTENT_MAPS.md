# v0.18：Markdown 内容地图与二级页面

## 目标

让“技术 · 工具 · 阅读 · 自然 · 生活”五个一级栏目在内容增多后仍然可维护：一级栏目负责入口和最近内容，二级地图负责方向，二级页面负责长期积累。

## 内容位置

```text
src/content/writing/                 文章（Markdown / MDX）
src/content/section-pages/           二级页面、站外专题、静态说明（Markdown / MDX）
src/content/tools.json               需要结构化排序和链接的工具条目
src/content/books.json               书架状态数据
src/content/nature.json              自然观察的结构化数据
```

## 二级地图

```text
技术：技术文章 / PCIe 与高速互连 / 芯片、固件与 SoC / 系统与开发环境 / 工程与职业
工具：自己 DIY 项目 / Chrome 扩展与网页工具 / PCIe 硬件工具 / 文字图片与记录 / 资料库
阅读：阅读文章 / 书架 / 我翻译的书 / 我写的书 / 站外专题
自然：植物 / 动物 / 季节 / 行走与观察 / 地方记忆
生活：生活与思考 / 运动 / 饮食 / 旅行 / 家庭 / 日常与方法
```

## 站外专题维护

直接编辑：

```text
src/content/section-pages/reading/reading-sites.md
```

保留六个二级标题；想把某个站点置顶时，直接移动它在该标题下的位置。名称是链接，说明尽量控制为一句；网页桌面端会最多显示两行。

## 规则

- 一级栏目只选一个；文章可同时拥有多个标签或专题。
- 没有真实公开内容的二级页可以保留简短说明，但不要虚构书目、路线、家庭信息或项目链接。
- 新增二级页时，创建对应 Markdown，填好 `section`、`routeSlug`、`title`、`description`、`order`、`kind` 和 `draft`。
- 运行 `./preview-local.cmd` 后检查地图链接、内容列表和移动端换行。
