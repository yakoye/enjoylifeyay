# v0.4.1: Node 24 environment alignment

## Changes

- Changed `package.json` engine requirement from Node 22-only to Node `>=24.0.0 <25`.
- Changed `.nvmrc` to `24`.
- Updated the lockfile root metadata and engine contract test.
- Updated deployment and handoff documentation for Node 24.
- Added a Windows-specific Rollup `EPERM` troubleshooting note.

## Reason

The local environment and Codex runtime use Node 24. Astro supports Node 22.12.0 or newer and supports even-numbered Node releases, so Node 24 is a suitable LTS target for this static Astro site.
