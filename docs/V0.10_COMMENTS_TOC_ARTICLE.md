# v0.10：评论、目录审计与新文章

## 完成内容

- 新增文章：`程序员、工程师与 AI 时代的独立创造`，发布日期为 2026-07-01。
  - 保留对 cnbang 2019-09-28 原文的链接；
  - 本站正文以阅读摘要与 2026 年补记为主，不转载对方原文全文。
- 每篇公开文章正文末尾增加极简评论区。
  - 无头像；
  - 无卡片、无背景块、无外框；
  - 仅保留昵称（可选）、留言、提交评论；
  - 评论默认审核后公开。
- 新增 Cloudflare Pages Functions API：`functions/api/comments.js`。
- 新增 D1 建表文件：`database/comments.sql`。
- 新增评论部署和审核说明：`docs/COMMENTS_D1.md`。
- 目录入口仍显示为“目录”，默认折叠。
- 目录条件从“至少两个标题”改为“至少一个二级至四级标题”。
- 对现有公开文章进行目录审计，并为原先无标题结构的文章补上语义标题：
  - 《平面国》总结；
  - 《臣服实验》读书笔记；
  - 方言：常州话；
  - PCIe 问题汇总 1。
- 新增命令：

  ```powershell
  npm run audit:toc
  ```

  它会检查每一篇 `draft: false` 的写作文章是否至少包含一个 `##`、`###` 或 `####` 标题。任何公开文章缺标题时，命令会失败；这样不会再出现公开文章没有“目录”的情况。

## 已知前提

评论区的网页界面会随静态站点一起发布，但真正保存评论前，需要你按 `docs/COMMENTS_D1.md` 创建 D1、导入 `database/comments.sql` 并绑定 `COMMENTS_DB`。
