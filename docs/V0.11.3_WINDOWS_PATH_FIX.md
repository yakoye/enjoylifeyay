# v0.11.3：Windows 路径修复

## 修复内容

Windows 下，Node 的 `file:` URL 的 `.pathname` 会以 `/C:/...` 的形式返回。旧版测试与目录审计脚本把该值继续传给 `node:path.join()`，会生成错误路径：

```text
C:\C:\Users\color\Documents\EnjoyLife\...
```

因此 `npm test` 中的目录测试会报 `ENOENT`，一键预览脚本也会在 `npm run audit:toc` 阶段停止。

本版已将以下文件改为使用 `fileURLToPath()`，把 `file:` URL 正确转换为 Windows 本地路径：

- `tests/v10-comments-toc-contract.test.mjs`
- `scripts/audit-writing-toc.mjs`

## 覆盖后如何验证

无需重新安装依赖；覆盖文件后在项目根目录执行：

```powershell
npm test
npm run audit:toc
```

两项通过后，再运行一键本地预览：

```powershell
.\preview-local.cmd
```

若刚覆盖完整源码包、或 `node_modules` 不存在，则使用：

```powershell
.\preview-local.cmd -Install
```
