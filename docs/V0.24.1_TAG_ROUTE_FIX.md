# v0.24.1 标签路由构建修复

## 问题

标签 `MSI / MSI-X` 含有 `/`。此前 `getStaticPaths()` 直接把原始标签作为动态路由参数，Astro 会把 `/` 当作路径分隔符，从而在预渲染 `/tags/MSI / MSI-X` 时报告 `Missing parameter: tag`。

## 修复

- 动态路由参数先通过 `encodeTagParam()` 转换为安全的单个路径段。
- 标签原文通过静态路由 `props` 传入页面，用于标题、筛选和正文展示。
- `tagHref()` 与 `getStaticPaths()` 统一使用同一套编码规则。
- 普通中文、英文标签保持原有可读性；路径保留字符使用 `~XX` 形式转义。

例如：

```text
MSI / MSI-X → MSI ~2F MSI-X
```

生成链接时空格仍会由 URL 正常编码，因此该标签不会再被拆成多级目录。
