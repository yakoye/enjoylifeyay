# v0.11.2：一键本地检查与预览

这版新增 Windows PowerShell 本地预览脚本。它把日常查看网站前需要的检查、构建、启动预览服务器和打开浏览器合并为一个动作。

## 最简单的使用方式

在资源管理器中双击仓库根目录的：

```text
preview-local.cmd
```

或在 PowerShell 的仓库根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\local-preview.ps1
```

也可以使用 npm 命令：

```powershell
npm run preview:local
```

脚本会自动执行：

```text
npm run check
npm test
npm run audit:toc
npm run build
npm run check:links
npm run preview -- --host 127.0.0.1 --port 4321
```

成功后会自动打开：

```text
http://127.0.0.1:4321/
```

## `npm ci` 什么时候执行

每次改 Markdown、图片、CSS 或页面组件时，不需要重新安装依赖。脚本发现 `node_modules/.bin/astro.cmd` 已存在时会跳过 `npm ci`，减少等待时间和 Windows 原生模块锁文件风险。

以下情况使用“完整安装并预览”：

```powershell
preview-local.cmd -Install
```

或：

```powershell
npm run preview:local:install
```

它会先执行：

```powershell
npm ci
```

再继续所有检查、构建和预览步骤。

适合：

- 第一次打开仓库；
- 刚运行过清理脚本；
- `package.json` 或 `package-lock.json` 有修改；
- 本地依赖丢失；
- 需要确认锁文件中的依赖可以完整安装。

## 停止本地预览

预览服务器会在后台运行。关闭浏览器不会停止它。

双击：

```text
stop-local-preview.cmd
```

或执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\stop-local-preview.ps1
```

也可以使用：

```powershell
npm run preview:local:stop
```

## 其他参数

只验证并构建、不启动预览：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\local-preview.ps1 -NoPreview
```

启动预览但不自动打开浏览器：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\local-preview.ps1 -NoBrowser
```

改用其他端口：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\local-preview.ps1 -Port 4322
```

## 常见问题

### `EPERM`、`rolldown-binding...node` 或 `rollup...node` 文件被占用

先执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\reset-local.ps1 -StopAllNode
```

然后重新运行：

```powershell
preview-local.cmd -Install
```

### 端口 4321 已被占用

若是本脚本启动的旧预览，先执行：

```powershell
stop-local-preview.cmd
```

如果仍然被其他软件占用，可以改用：

```powershell
preview-local.cmd -Port 4322
```

### 评论区显示“评论服务暂时不可用”

这是正常的。`local-preview.ps1` 使用 Astro 静态预览，不会启动 Cloudflare Pages Function 或连接生产 D1。

需要测试评论 API 时，请使用：

```powershell
npm run build
npx wrangler pages dev dist --d1 COMMENTS_DB=<你的D1数据库ID>
```

详见 [`COMMENTS_D1.md`](COMMENTS_D1.md)。

### Windows 路径说明（v0.11.3）

`npm test` 与 `npm run audit:toc` 已使用跨平台的 `fileURLToPath()` 处理文章目录。若你看到 `C:\C:\Users\...` 这种重复盘符错误，说明仍在使用 v0.11.2 或更早版本的测试/审计脚本；请覆盖 `tests/v10-comments-toc-contract.test.mjs` 与 `scripts/audit-writing-toc.mjs`，或直接使用 v0.11.3。
