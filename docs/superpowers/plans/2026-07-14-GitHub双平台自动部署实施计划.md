# GitHub 双平台自动部署实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将新目录整理为唯一源码仓库，移除评论和 D1，修复两篇文章，并让 `main` 推送自动部署到 Cloudflare Pages 与 GitHub Pages。

**Architecture:** 保留现有 Cloudflare Direct Upload 项目及 `https://enjoylifeyay.goldke.online` 自定义域名，由 GitHub Actions 在验证成功后使用 Wrangler 发布根路径构建；GitHub Pages 使用同一提交和 `/enjoylifeyay/` 基础路径生成备用站。GitHub Pages 构建结束后统一修正 Markdown、静态资源和 Pagefind 产生的根路径，canonical 始终指向 Cloudflare 自定义正式域名。

**Tech Stack:** Astro 7、Markdown/MDX、Node.js 24、Node Test Runner、Pagefind、GitHub Actions、Wrangler 4、Cloudflare Pages。

---

## 文件职责

- `.github/workflows/deploy.yml`：验证、Cloudflare 发布与 GitHub Pages 发布。
- `astro.config.mjs`：读取 `BASE_PATH` 并配置 Astro 静态构建基础路径。
- `scripts/prefix-base-path.mjs`：只在 GitHub Pages 构建时修正生成文件中的根路径。
- `scripts/verify-science-references.mjs`：核对科学文章引用编号、PMID 和 DOI。
- `tests/v22-publishing-contract.test.mjs`：锁定无评论、无过期文件、双部署和 README 约定。
- `tests/v10-comments-toc-contract.test.mjs`：保留目录测试，反向验证评论功能已经退出。
- `src/layouts/BaseLayout.astro`：保持 Cloudflare canonical，并让快捷搜索地址兼容基础路径。
- `src/layouts/ArticleLayout.astro`：移除评论组件。
- `src/pages/search/index.astro`：让 Pagefind 脚本和搜索结果兼容 `/enjoylifeyay/`。
- `README.md`：唯一开发目录、日常内容更新、双平台自动发布和故障处理。
