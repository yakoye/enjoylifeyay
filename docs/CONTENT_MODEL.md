# 内容模型

本站用较少、边界清楚的元数据组织文章。

## 文章只维护两项分类

```yaml
section: technology/pcie
tags: ["PCIe", "Linux", "MSI / MSI-X"]
```

- `section` 是文章唯一的内容归属，决定它位于哪个一级栏目与二级栏目。
- `tags` 是可跨栏目检索的具体主题。每篇最多 3 个，只写读者会主动寻找的词。

不再维护 `domain`、`format`、`topics` 三套重复字段。

## section：纵向内容地图

```text
技术：技术文章 / PCIe 与高速互连 / 芯片、固件与 SoC / 系统与开发环境 / 工程与职业
工具：软件与效率
阅读：阅读文章
自然：植物 / 地方记忆 / 行走与观察
生活：思考 / 运动 / 饮食 / 旅行 / 家庭 / 日常与方法
```

文章只选择一个 `section`。读者应能在这个位置自然找到它。

## tags：横向主题索引

标签适合：`PCIe`、`Linux`、`Git`、`命令行`、`C 语言`、`Word`、`Excel`、`速查表`、`骑行`、`跑步`、`植物`。

标签不适合：`技术`、`生活`、`记录`、`随笔`、`感悟`、`程序员`等过宽或只为单篇文章服务的泛词。

标签只有关联至少两篇公开文章时，才会出现在 `/tags/` 并生成独立页面；单篇文章里的标签仍可保留为普通文字，不制造空的主题入口。

## 系列

同一领域不等于系列。只有已经形成至少三篇、具备明确总标题和连续阅读关系的文章，才值得做成一个独立的 Markdown 系列页。

当前没有启用中的系列元数据。旧的系列 URL 会跳转到对应二级栏目，不影响旧书签。
