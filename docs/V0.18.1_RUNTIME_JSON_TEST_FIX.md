# v0.18.1：覆盖升级兼容修复

## 问题

在 Windows 上以“解压覆盖”的方式升级时，旧文件 `src/content/reading-sites.json` 不会自动删除。
旧测试错误地要求该文件必须不存在，因此即使站点已经只通过 Markdown 维护站外专题，测试仍会失败。

## 修复

测试现在验证运行时页面和内容配置不会引用旧 JSON，也不再要求物理删除遗留文件。
站外专题的唯一运行时维护文件仍是：

```text
src/content/section-pages/reading/reading-sites.md
```

## 本地验证

```powershell
.\preview-local.cmd
```

不需要重新 `npm ci`，除非依赖确实损坏或被清理。
