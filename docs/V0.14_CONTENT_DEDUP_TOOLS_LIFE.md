# v0.14：内容去重、工具重组与生活页面

## 本次目标

- 删除旧 旧个人站 站点与源码仓库两条长期收藏，避免重复历史入口。
- 阅读页只保留公开阅读文章，取消书架、读过和专题的重复列表。
- 技术页只显示有公开文章的专题；空专题不再生成公开详情页或链接。
- 工具页按“自己 DIY 项目 → Chrome 扩展与网页工具 → PCIe / 硬件工具 → 文字、图片与记录工具 → 资料库 → 历史内容来源”排序。
- 饮食、影集与家庭、运动网站移到生活页，不在工具页重复展示。

## 以后维护

### 新工具或自己 DIY 项目

编辑 `src/content/tools.json`：

- `diy-project`：自己 DIY 项目。
- `browser-extension`：Chrome 扩展与网页工具。
- `pcie-hardware`：PCIe / 硬件工具。
- `writing-media`：文字、图片与记录工具。
- `knowledge-library`：资料库。
- `websites-life`：生活页的饮食、影集与家庭、运动网站。

### 避免空专题

只有文章 Frontmatter 中的 `series` 指向某专题，且文章 `draft: false`，该专题才会出现在技术页并生成专题详情页。

### Rich Editor 与 quick_note_richtext

在 `tools.json` 中：

- `url` 用于直接打开可渲染的 HTML 工具。
- `githubUrl` 用于保留源码位置。

不要把 GitHub `blob` 地址放进 `url`，否则访客看到的是代码查看页而不是工具。
