# v0.20：Markdown 渲染与列表一致性

## 目标

本版将面向读者的静态页面文字、二级栏目说明与站外专题继续收拢到 Markdown 内容文件中，并让 Markdown 列表与数据目录使用同一套可见样式。

## 内容位置

| 内容 | 维护位置 |
| --- | --- |
| 一级页面标题、说明与短引言 | `src/content/site-pages/*.md` |
| 关于、关于我等完整静态页面正文 | `src/content/site-pages/*.md` |
| 二级栏目标题、说明与正文 | `src/content/section-pages/<section>/*.md` |
| 站外专题 | `src/content/section-pages/reading/reading-sites.md` |
| 博客文章 | `src/content/writing/*.md` 或 `.mdx` |

主页与“现在”页共享 `src/content/site-pages/home.md` 中的当前记录，避免出现两套相近文案。

## 列表规则

- Markdown 的 `-` 会保留为 HTML `ul / li` 语义。
- 普通页面目录、文章中的无序列表、Markdown 二级页面和数据驱动目录，统一显示为较大的黑色圆点 `•`。
- 站外专题桌面端每条说明最多显示两行；移动端自然换行。
- 日期时间线保持时间线结构，不改造成普通圆点列表。

此前站外专题为了限制两行文字而把 `li` 变为 `display: -webkit-box`，部分浏览器因此不显示列表标记。本版改为保持列表项语义，再用高度限制两行内容。

## 代码块明暗主题

Astro 使用 Shiki 同时生成浅色和深色 token。浅色主题使用浅灰代码背景，深色主题才使用深色代码背景；主题切换时，代码文字和背景会一起切换。
