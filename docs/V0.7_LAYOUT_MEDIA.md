# v0.7：居中导航、归档入口与图片管理

## 已完成

- 顶栏改为三栏 Grid：品牌固定在左、导航以视口中心为基准居中、搜索与主题按钮固定在右。
- 主导航增加 `/archive/`，顺序为“项目 · 归档 · 关于”。
- 采用提供的 favicon 生成浏览器、Apple Touch 与 manifest 图标资源；HTML 同时声明 ICO 与 PNG 图标，并通过版本参数降低旧缓存影响。
- `/archive/` 是唯一使用“左日期 / 右标题”的文章时间线。写作、专题、工具、项目等页面维持文本目录布局。
- 移除首页标题下、专题页、写作筛选等装饰性横线；收紧模块间距，避免 `h2` 默认上边距和 section 间距叠加产生大空白。
- 写作筛选使用 `data-writing-filters` 和内联初始化脚本。领域与形式筛选会隐藏不匹配条目、更新当前选中样式和 URL Query 参数。
- 专题统计由“`2 篇已公开文章`”改为“`（2篇）`”。
- 新增 `mediaKey` 内容字段与 `npm run import:legacy-images`；方便把旧 EnjoyLifeBlog 图片转为本站 `/images/legacy/...` 路径。

## 未自动执行的事情

本打包环境无法替站长访问旧 GitHub 博客下载二进制图片，因此本版没有把远程旧图直接写进压缩包。项目内已经提供了可本地执行的迁移脚本和完整说明。运行 `npm run import:legacy-images -- --write` 后，图片会进入 `public/images/legacy/`；接着构建并通过 Wrangler 发布，就会由 Cloudflare Pages 提供。
