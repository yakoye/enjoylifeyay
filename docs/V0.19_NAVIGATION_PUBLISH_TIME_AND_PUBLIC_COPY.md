# v0.19：二级地图返回、发布时间与公开文案

## 二级地图

技术、工具、阅读、自然、生活的父页与二级页都使用同一套地图。

```text
技术：技术文章 · PCIe 与高速互连 · 芯片、固件与 SoC · …
```

其中一级栏目名始终是父页链接。例如正在阅读“技术文章”时，点击“技术”即可返回 `/technology/`；其他四个栏目同样适用。

二级页继续由以下 Markdown 文件维护：

```text
src/content/section-pages/<section>/
```

## 文章发布时间

每篇文章现在都有两个时间字段：

```yaml
date: 2026-07-05
publishedAt: 2026-07-05T12:00:00+08:00
```

- `date`：首次公开日期，供列表、归档与排序使用。
- `publishedAt`：文章正文顶部显示的准确发布时间，显示为 `发布：YYYY-MM-DD HH:mm:ss`。
- `updated`：有实质性修改时填写；详情页继续只显示最后修订日期。

旧文若只保留了日级日期，先使用该日 `00:00:00+08:00`。获得原始发布时间后，直接修改 `publishedAt` 即可。

## 公开页面文案

公开页面只写给读者：内容、阅读线索、作者与站点本身。

开发、迁移、数据、私有资源、部署和维护说明只写入 `docs/`。发布前运行：

```powershell
npm run audit:public-copy
```

它会阻止常见的开发者占位语出现在公开页面中。
