# v0.15：旧个人站本地化与公开隔离

## 已完成

- 旧个人站的 14 篇文章均迁入 `src/content/writing/`；其中原先的两个草稿已整理为可公开的本地文章。
- 文章页面不再展示旧站来源、原文入口或迁入日期。
- 旧文章中的图片路径改为本站 `/images/articles/<mediaKey>/...` 本地路径。
- 归档页不再列出旧个人站条目；本站文章直接进入本站正文，未迁入的历史技术条目仅保留 CSDN 原始页面。
- `C语言速查` 已从历史存档移除。
- `scripts/legacy-assets.private.json` 只用于一次性下载旧图，已被 `.gitignore` 排除，不应提交到 GitHub，也不会部署到 Cloudflare Pages。

## 第一次升级后必须做的事

在项目根目录运行：

```powershell
.\preview-local.cmd -FetchLegacyAssets

# 或分步执行
npm run fetch:legacy-assets
npm run verify:legacy-assets
```

成功后，图片会写入 `public/images/articles/`。这些图片会随静态构建部署到 Cloudflare Pages；访客不会再请求旧个人站。

确认无误后提交图片：

```powershell
git add public/images/articles
git add -u
git commit -m "content: localize legacy article images"
```

`legacy-assets.private.json` 不要提交；它包含仅用于本地迁移的旧图片地址。

## 发布前检查

```powershell
npm run verify
```

如果图片下载尚未完成，`npm run verify:legacy-assets` 会明确列出缺失文件，阻止带有缺图的正式发布。
