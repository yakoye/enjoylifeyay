# v0.14.1 本地预览与升级可靠性修复

## 修复内容

1. 修复 v0.14 测试对 `src/content/projects.json` 的物理文件存在性要求。

   Windows 上通过“解压并覆盖”升级时，旧版本的未引用文件可能保留在目录中。现在测试只验证项目集合已经从 Astro 内容模型中移除，并保留旧 `/projects/` 跳转兼容；不会因为一个未被加载的遗留 JSON 文件而失败。

2. `preview-local.cmd -Install` 现在会自动：

   - 停止残留 `node.exe`；
   - 删除 `node_modules`、`dist`、`.astro`；
   - 重试删除目录；
   - 再执行 `npm ci`。

   这样可以显著减少 Windows 下 Rollup / Rolldown 原生 `.node` 文件被占用导致的 `EPERM`。

## 使用

普通内容改动：

```powershell
.\preview-local.cmd
```

覆盖新版本、修改依赖或 `npm ci` 报错后：

```powershell
.\preview-local.cmd -Install
```

若 Windows 仍然提示 `EPERM`：

1. 关闭 VS Code 和所有打开项目目录的资源管理器窗口；
2. 重启 Windows；
3. 首先运行：

```powershell
.\preview-local.cmd -Install
```

不要在每次仅修改 Markdown、JSON、CSS 时使用 `-Install`。
